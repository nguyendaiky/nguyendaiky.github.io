Tìm hiểu Git/Github cơ bản

By Kỳ Nguyễn

November 11, 2021

Git là gì?

Git là một “hệ quản trị phiên bản” (version control) được phát triển bởi Linus Torvalds.

Khi lập trình viên làm việc, thì họ sẽ liên tục cập nhật source code, dù cho dự án đó đã hoàn thành thì vẫn phải update các phiên bản mới cho phần mềm, sửa lỗi và bổ sung thêm các tính năng khác… Hơn hết, khi làm việc nhóm cùng đội của mình. Mỗi người sẽ được giao làm một việc nhỏ, gộp lại thành một dự án lớn. Source code sẽ được chia ra làm nhiều phần, nhiều mô-đun nhỏ cho mỗi người làm. Vì vậy, việc quản lí tiến độ dự án, khắc phục lỗi, chạy thử… sẽ rất khó khăn và phức tạp.

Do đó, nhờ có “hệ quản trị phiên bản” Git sẽ giúp giám sát những thay đổi của source code, bằng cách lưu nhiều phiên bản khác nhau của một mã nguồn, được nhân bản (clone) từ một kho chứa mã nguồn (repository). Có thể hiểu đơn giản là nó lưu lại các phiên bản của những lần thay đổi source code, có thể khôi phục lại dễ dàng trong những lần sau đó. Và những người khác cũng có thể xem các thay đổi của mình trong những lần thay đổi đó, có thể gộp phiên bản của mình vào phiên bản của họ. Sau đó đưa tất cả lên kho chứa mã nguồn chung.

Github là gì?

Có thể nói Github là một “mạng xã hội” dành riêng cho lập trình viên. Đây là nơi cho phép các lập trình viên tương tác, trao đổi với nhau, theo dõi, đóng góp (contribute) và thả sao (star) cho những dự án mà bạn cảm thấy thú vị. Dự án có càng nhiều star cho thấy mức độ quan tâm và sự hữu ích mà nó đóng góp cho cộng đồng. Hoặc khi tham gia contribute vào một dự án mà bạn cảm thấy thú vị, bạn sẽ được cộng tác với những lập trình viên giỏi đến từ khắp nơi trên thế giới, từ đó học được rất nhiều điều từ họ. Hơn hết nếu dự án thành công, tên tuổi của bạn sẽ được gắn liền với dự án đó. Góp phần làm đẹp profile, CV của bạn cho công việc sau này.

Một số bạn sẽ nhầm lẫn giữa 2 khái niệm Git và GitHub. Git là công cụ để track sự thay đổi trong source code. GitHub là website nơi lưu trữ tất cả các repository chứa source code trên server online. Bởi vì online nên dễ dàng cho làm việc nhóm, làm việc từ xa hoặc có thể tổ chức các project thành portfolio để show cho nhà tuyển dụng. Vì đa số các công ty lập trình đều quản lý dự án bằng Git, đôi lúc sẽ yêu cầu ứng viên có kiến thức về Git, vì vậy việc nắm rõ kiến thức về Git sẽ giúp ích rất nhiều.

Cơ chế hoạt động

Cơ chế hoạt động của git/github được chia thành 2 phần:

  • Remote repository
  • Git structures

Remote repository đóng vai trò như là một kho lưu trữ trên internet (remote), đó là Github.

Git structures là cái chúng ta cài đặt ở máy của mình (local), đó là Git.

Git structures chia thành 3 phần:

  1. Workspace/Working tree
  2. Staging Area/Index
  3. Local repository

Khi làm việc với Git thì các file của chúng ta sẽ có 2 trạng thái chính đó là Tracked và Untracked:

  • Untracked: là trạng thái của các file mới được tạo ra, chưa được Git đánh dấu để theo dõi.

  • Tracked: là trạng thái các file đã được Git đánh dấu để theo dõi. Được chia thành 3 trạng thái phụ:

    • Unmodified: không có thay đổi.
    • Modified: đã có thay đổi nhưng chưa thêm vào Index.
    • Staged: đã thêm vào Index và sẳn sàng cho commit.

    Lưu ý: một file phải được thêm vào Index và có trạng thái Staged thì mới có thể commit.

Sau khi commit các file sẽ được chuyển vào local repository. Lúc này các file đó đã sẳn sàng để push lên remote repo trên Github.

Cách thiết lập

Đầu tiên chúng ta truy cập vào github.com, nhấn Signup để đăng ký tài khoản. Nếu bạn đã có tài khoản thì nhấn Signin để đăng nhập vào.

Sau khi đăng ký và đăng nhập vào tài khoản, ta sẽ thấy giao diện Github như sau:

Tiếp theo truy cập git-scm.com/downloads, chọn đúng phiên bản cho hệ điều hành của mình. Tải về và cài đặt vào máy.

Sau khi cài đặt xong, ta kiểm tra đã thành công hay chưa bằng Command Prompt/Terminal.

Để mở Command Prompt trên Windows, ta nhấn tổ hợp Windows + R, gõ cmd. Để mở Terminal trên Mac/Linux, ta nhấn Ctrl + Alt + T. Sau đó gõ git –version để kiểm tra, nếu hiện ra phiên bản git hiện tại thì đã cài đặt thành công.

Ta cũng tiến hành config git ở local bằng email và username đã đăng ký github.

git --version
# git version 2.32.0
git config --global user.email "your-email@mail.com"
git config --global user.name "your-name"

Cách sử dụng

Git

  • Git init
git init
# Initialized empty Git repository in ...

Lệnh git init sẽ tạo một local reposity ngay thư mục hiện tại. Hiển thị file ẩn lên, ta thấy một folder .git đã được tạo ra, trong đó chứa các thông tin và thiết lập cho reposity.

  • Git add
git add filename.txt
git add .

Lệnh git add thay đổi trạng thái file thành Tracked và thêm file đó vào Index để tiến hành theo dõi sự thay đổi của file. Lệnh git add filename.txt để thêm file cụ thể, lệnh git add . để thêm tất cả các file.

  • Git restore
git restore --staged <file>

Lệnh git restore dùng để unstage một file (có thể hiểu là ngược lại với lệnh git add).

  • Git status
git status

Lệnh git dùng để kiểm tra trạng thái các file trong local repository. Các file trong repository được git theo dõi theo các trạng thái khác nhau. Màu xanh là các file ở trạng thái stage và đã thêm vào Index, sẳn sàng cho việc commit. Màu đỏ là các file untracked, modified, deleted, cần phải dùng lệnh git add thêm vào Index để commit chúng.

  • Git commit
git commit -m 'your-message'

Lệnh git commit lưu các thay đổi vào local repository. Ta có thể dùng lệnh git commit -a để thực hiện add và commit các thay đổi cùng lúc vào local repository.

  • Git push
git push origin [branch-name]

Lệnh git push đẩy file đã commit từ local repository lên remote repository (Github).

  • Git clone
git clone [link-to-repository]

Lệnh git clone dùng để tạo một bản clone của remote repository (nói dễ hiểu là down repository về máy). Đây là lệnh dùng rất thường xuyên khi tìm kiếm mã nguồn mở trên mạng. Có 2 cách để tải dự án về máy, một là download bằng file zip, hai là dùng git clone.

  • Git remote
git remote add [name] [link-to-remote-repository]

Lệnh git remote dùng liên kết local repository và remote repository.

  • Git pull
git pull origin [branch-name]

Lệnh git pull tải các file ở remote repository đã liên kết về local repository (có thể hiểu là ngược lại với git push).

  • Git branch
git branch

Lệnh git branch dùng để xem nhánh làm việc hiện tại. Để tạo nhánh mới và xóa nhánh ta dùng các lệnh bên dưới:

git branch [branch-name] # create
git branch -d [branch-name] # delete
  • Git switch
git switch [branch-name]

Lệnh git switch dùng để chuyển sang nhánh làm việc mới.

  • Git merge
git merge [branch-name]

Lệnh git merge gộp nhánh branch-name vào nhánh làm việc hiện tại.

  • Git log
git log

Lệnh git log dùng để xem lịch sử các commit của repository cho đến hiện tại. Ta thêm —oneline để xem ngắn gọn và thêm —summary để xem chi tiết.

Github

  • Tạo repository

Ở trang chủ Github, ta nhấn New. Đặt tên repository, mô tả, trạng thái, Readme và giấy phép.

  • Commit

Để thay đổi nội dung một file ta chọn file. Nhấn edit, sửa file. Sau đó điền commit message rồi nhấn Commit. Lệnh này sẽ tiến hành thay đổi file và lưu vào repo.

  • Request

Cho biết những thay đổi trong source code, và yêu cầu owner của source code xem xét nó và merge nó vào master branch.

  • Merge

Lệnh merge này cho phép bạn hợp nhất những thay đổi vào một branch.

Trang trí github

Tạo repo có tên trùng với username của bạn, add file README và thực hiện trang trí bằng markdown.

Các bạn tham khảo các file README tại Awesome-Profile-README-templates và làm một trang cho mình.
